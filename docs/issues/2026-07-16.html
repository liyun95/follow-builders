把团队知识编码成 agent 可用的基础设施

Boris Cherny 认为，工程团队过去通过 lint、CI、测试和脚本放大个人产出；在 agent 时代，这类自动化更重要，因为它会同时加速一整支 agent 队伍。他主张把领域知识写进 CLAUDE.md、REVIEW.md、skills、docs、代码注释和 memories，让 agent 在没有额外上下文的情况下也能高质量贡献代码。

Aaron Levie · ceo @box

企业 agent 采用的关键：流程、数据、权限与 headless 软件

Aaron Levie 总结与大型企业 IT 负责人讨论 agent 采用的要点：变革管理仍是核心，企业需要把结构化和非结构化数据整理到适合 agent 的形态；越来越多 IT 团队把工程师嵌入业务流程，类似内部 FDE；跨职能 workflow 的权限与数据建模很复杂，agentic systems 可能需要自己的角色与权限。他还指出，企业软件未来需要更 headless，否则会被 agent 集成体验和成本拖累。

Thariq · Claude Code @anthropicai

Claude Code builder 的 prompt 经验：thin prompts、thick artifacts

Thariq 给出简洁的 prompting 公式：thin prompts、thick artifacts + context、thin skills。也就是说，不把复杂性堆在 prompt 本身，而是把可复用上下文、产物和技能设计好，让模型在更稳定的结构中工作。
